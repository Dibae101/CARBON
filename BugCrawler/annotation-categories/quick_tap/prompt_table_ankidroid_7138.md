# Example prompt used for AnkiDroid run
Issue: https://github.com/ankidroid/Anki-Android/issues/7138

| Category | Prompt Content |
|----------|---------------|
| **Question** | "I need your assistance in reproducing bug reports for Android UI testing on the emulator. Our goal is not just to follow the steps leading to where the bug occurs in the app, but also to verify that the buggy behavior specified in the bug report is indeed triggered. During the initial prompt, I will give you the app name, bug report, and details about the app screen. Along with each prompt, I will also provide a screenshot of the current device screen. Use the screenshot to understand the visual state of the app — this helps you identify colors, visual elements, hidden controls, notification content, permission dialogs, and elements that may not appear in the UI hierarchy text. The screenshot is your UIAutomator Viewer — use it to SEE the screen just as a human tester would. Your role is to provide suggestion at a time." |
| **Rules** | "Your suggestion will be presented as a list, Suggestion:[a1], [a1, a2], etc. Each suggested action should be a dictionary with keys such as 'action' indicating the intended action, 'feature' specifying the feature of the target widget... Actions available: 1.'click' 2.'long_click' 3.'set_text' 4.'orientation' 5.'swipe' 6.'back' 7.'scroll' 8.'home' 9.'open_notifications' 10.'tap_screen' (requires x,y coordinates) 11.'swipe_region' (requires start_x,start_y,end_x,end_y) 12.'pinch' (pinch_type 'in' for zoom in, 'out' for zoom out) 13.'two_finger_swipe' 14.'two_finger_tap' 15.'three_finger_swipe' 16.'drag_and_drop' (CRITICAL: start from drag handle, use hold_ms=200 for reorder) 17.'rotate_gesture' 18.'quick_tap' (timing-sensitive, minimal delay). IMPORTANT: For quick_tap use delay_ms parameter to control timing. Use batched actions [click, quick_tap] to fire actions back-to-back without LLM loop delay." |
| **Test sequence** | **Step 0 input to LLM:** []. *Current Screen Information: #Current Activity: .DeckPicker. # UI Information: click has the following group(s): 1#.toolbar:[...AnkiDroid, '34 cards due (9 min)']; 2#.['DeckPickerHoriz']; ... Bug triggered!* |
